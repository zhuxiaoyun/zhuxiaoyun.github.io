  2. 引用类型的装箱和拆箱

例1：
Boxing and Unboxing
Integer a = 2; //Boxing 自动装箱操作，把2装进了箱子, a 指向箱子
Integer b = new Integer(2); //Boxing，同上，b指向箱子
int v = a.intValue(); //Unboxing 手动拆箱操作，把数值拿出来
int c = 2;
Integer d = Integer.valueOf(2); //手动装箱，把int型转换为Integer，若在-128到127范围，直接返回该对象的引用，否则在堆中重新new 一个。

System.out.println(a == b); // false 两个引用没有引用同一对象
System.out.println(a == c); // true a自动拆箱成int类型再和c比较，拆箱的过程，因为等号一边存在基本类型所以编译器后会把另一边的Integer对象拆箱成int型，这时等号两边比较的就是数值大小
System.out.println(a == d); //true

new Integer(2) == 2 //true 自动拆箱比较
new Integer(2) == new Integer(2) //false 没有引用同一对象
Integer.valueOf(2) == Integer.valueOf(2) //true 指向同一缓存中的数据
Integer.valueOf(1000) == Integer.valueOf(1000) //false ，超出数值，重新new对象空间
Integer.valueOf(2).intValue() == 2 //true 拆箱操作，数值比较
new Integer(2).equals(new Integer(2)) //true 数值比较

例2：
Integer f1 = 100, f2 = 100, f3 = 150, f4 = 150;
System.out.println(f1 == f2); //true
System.out.println(f3 == f4); //false
==运算比较的不是值而是引用，装箱的本质，当我们给一个Integer对象赋一个int值的时候，会调用Integer类的静态方法valueOf，如果整型字面量的值在-128到127之间，那么不会new新的Integer对象，而是直接引用常量池中的Integer对象

类型转换

整数的默认类型是 int
小数的默认类型是 double

  1. Java自动类型转换和强制转换
    1）两种类型是彼此兼容的
    2）转换的目的类型占得空间范围一定要大于转化的源类型
    　　正向过程：由低字节向高字节自动转换
    　　byte->short->int->long->float->double
    　　逆向过程：使用强制转换,可能丢失精度。
    　　int a=(int)3.14;
  1. 隐含强制类型转换

b += 2 相当于 b = (short)(b+2);
b+=2是可以编译成功的。+=也是一个赋值运算符，在底层是一个动作并且有一个强制转换的一个过程（不用我们自己强转）；
b=b+2;不能编译成功，这是两个动作，先求和再赋值，求和时b自动类型提升，求和结果是int类型，但是因为编译器无法判断数值大小，所以无法为我们强制转换类型，所以编译报错

总结：
(1)如果右边是常量，编译器会判断数值大小是否在范围内，如果在就做强制类型转换，不在就报错；

(2)如果右边是变量，编译器无法判断数值大小，不会做强制类型转换，会报错（可以自己强制转换）；

(3)+=，-=，*=，/=,%=赋值运算符不管是否在范围内都会做强制类型转换。

运算符

  1. j=j++, j=++j, j++ 和 ++j

j=j++, 会先保存j的值，再j++，此时确实j增加了，但是最后又把保存的值赋值给了j，所以j的值不变
j=++j, 先把j的值增加，此时j值增加了，再保存j的值，再把保存的值赋值给j，所以j值增加
j++和++j，j的值都增加

equals的重写，和hashCode解析

equals ,比较含有多个成员变量的对象时需要重写

覆盖equals时需要遵守约定：
（对于任何非null的引用值x和y）
自反性，x.equals(x) = true
对称性，x.equals(y) = true, 则y.equals(x) = true
传递性，x.equals(y) = true, y.equals(z), 则x.equals(z)
一致性，多次调用x.equals(y)都一致的返回true，或者false
覆盖equals时总要覆盖hashCode

== 比较值类型的值是否相等
== 比较引用类型是否为同一个对象
equals 比较引用类型的值是否相等
hashCode -> 返回对象存储地址转化的哈希值，如果重写equals，必须重写hashCode
原因：
hashcode不相同，用equals()方法判断的返回的一定为false。
hashcode相同，equals()方法返回值不能确认，可能为true，可能为false。
判断的时候先根据hashcode进行的判断，相同的情况下再根据equals()方法进行判断。如果只重写了equals方法，而不重写hashcode的方法，会造成hashcode的值不同，而equals()方法判断出来的结果为true。

toString

employee1 == employee2 ?
employee1.equals(employee2) ?
a.equals(b)=true ==>> a.hashCode() == b.hashCode()
(a.equals(b)=true, 必定a.hashCode() == b.hashCode()，反之不一定)

封装

封装，隐藏了类的内部实现机制，可以在不影响使用的情况下修改类的内部结构，同时也保护了内部数据。对外界而言，它的内部细节是隐藏的，暴露给外界的是它的访问方法。

可见性修饰符的作用范围：
private 类内部
(默认) 包内部
protected 派生类（子类），使用时要加注释
public 外部

尽量 只使用 private 和 public
每个文件里至多只能有一个public class,类名称和文件名一样，package 要和路径名一样
interface里方法不加任何可见性，就是public，规定的，和类不一样

派生类中修改封装可见性，面试中会涉及，但实际工作中不建议使用
private -> public ? 可以的
public -> private ? 不可以的
可以增加可见性，但不能减少可见性

多态

多态，程序中定义的引用变量所指向的具体类型和通过该引用变量发出的方法调用，在编程时并不确定，而在程序运行期间才确定
不修改源程序代码，就可以让引用变量绑定到不同的类实现上，从而改变程序运行时所绑定的具体代码，让程序可以选择多个运行状态，这就是多态性。

例子：
Manager extends Employee
Employee p = new Manager() //向上转型
一个父类可以引用不同的子类，只有在程序运行时才知道引用变量所指向的具体实例对象，如上，引用变量p是Employee类型，但程序运行时指向的是Manager类型

Java实现多态有三个必要条件：继承、重写、向上转型。
1.必须有子类继承父类
2.子类重写父类的方法
3.父类的引用指向了子类的实例
4.对象的行为和子类保持一致
参考链接 多态的解释与经典示例

重写和重载

重写：一般在父类中申明，子类中实现，继承中，在子类要实现的方法中，方法名、返回值、传的参数和父类的一样，但具体实现不一样
重载：一般在一个类中，有多个同名方法，但参数不一样（参数类型、参数个数、有无参数）。
